## 1. Introduksjon

Å gjøre nettsteder og applikasjoner brukbare for personer med kognitive og lærevansker påvirker alle deler av design og utvikling.

Tradisjonelt fokuserte tilgjengelighet på å gjøre grensesnittet brukbart for personer med sensoriske og fysiske funksjonsnedsettelser (syn, hørsel eller mobilitet). Noen tilgjengelighetsfunksjoner vil hjelpe personer med kognitive og lærevansker. Ofte inkluderer problemene som påvirker personer med kognitive og lærevansker:

- Design.
- Kontekst.
- Struktur.
- Språk.
- Brukervennlighet.
- Andre faktorer som er vanskelige å inkludere i generelle retningslinjer.

Noen vanlige design skaper barrierer for personer med kognitive og lærevansker. Mønstrene som presenteres er utformet for å unngå slike barrierer for personer med kognitive og lærevansker. Selv om denne veiledningen kan forbedre brukervennligheten for alle, er disse mønstrene viktige for at noen personer med kognitive og lærevansker skal kunne bruke innhold uavhengig.

## 2. Hovedtemaer

For å møte behovene til personer med kognitive og lærevansker har vi strukturert COGA W3C etter følgende hovedtemaer:

### [3.1 Mål 1: Hjelpe brukere å forstå hva ting er og hvordan de skal brukes](https://grunnmuren.obos.no/3-1-maal-1)

Brukere med kognitive og lærevansker kan ha problemer med orientering og læring. Dette kan føre til at folk blir desorienterte på et nettsted.

### [3.2 Mål 2: Hjelpe brukere med å finne det de trenger](https://grunnmuren.obos.no/3-2-maal-2)

Brukere med kognitive og lærevansker kan ha problemer med å finne innholdet de trenger. De kan også ha problemer med å orientere seg i innholdet eller oppgaven. Brukere bør raskt og enkelt kunne finne det de leter etter.

### [3.3 Mål 3: Bruk klart og forståelig innhold](https://grunnmuren.obos.no/3-3-mal-3)

Noen brukere har svekket språkforståelse. Flere av disse brukerne forstår innhold som bruker lettforståelig språk. For eksempel kan noen med språkvansker være i stand til å forstå enkle setninger og vanlige ord. Imidlertid kan komplekst språk med uvanlige ord være utilgjengelig for dem.

### [3.4 Mål 4: Hjelpe brukere med å unngå feil og vite hvordan de kan rette dem](https://grunnmuren.obos.no/3-4-maal-4)

Brukere skal kunne unngå feil og enkelt rette dem hvis det oppstår feil.

Det er vanskelig for mange brukere å fylle ut skjemaer, spesielt personer med kognitive og lærevansker.

### [3.5 Mål 5: Hjelpe brukerne med å fokusere](https://grunnmuren.obos.no/3-5-maal-5)

Distraksjoner kan hindre brukere med kognitive og lærevansker i å fullføre oppgavene sine.

Når brukere blir distrahert, kan det være vanskelig for dem å huske hva de holdt på med. Da kan de ikke lenger fullføre oppgaven. Dette er spesielt problematisk for brukere med både svekket oppmerksomhet og svekket hukommelse, for eksempel brukere med demens.

### [3.6 Mål 6: Sørg for at prosesser ikke er avhengige av minne](https://grunnmuren.obos.no/3-6-maal-6)

Minnebarrierer hindrer mange brukere i å bruke produkter eller få tilgang til hjelp eller innhold.

Personer med funksjonsnedsettelser som påvirker hukommelse eller språk kan synes det er vanskelig eller umulig å overvinne hukommelsesbarrierer.

For eksempel har mange brukere svekket korttidshukommelse. I gjennomsnitt kan folk huske informasjonsbiter samtidig. En person med svekket arbeidshukommelse kan være i stand til å huske én til fire informasjonsbiter samtidig.

### [3.7 Mål 7: Gi hjelp og støtte](https://grunnmuren.obos.no/3-7-maal-7)

Gjør det enkelt for brukere å få hjelp når de støter på problemer og gi tilbakemeldinger. Hvis brukere har problemer med å sende tilbakemeldinger, kan de ikke fortelle deg om de ikke kan bruke innholdet. Du vil ikke vite når de opplever problemer.

### [3.8 Mål 8: Støtte tilpasning og personalisering](https://grunnmuren.obos.no/3-8-maal-8)

Mange brukere trenger produkter som støtter tilpasning og personalisering. Brukere bør kunne bruke tillegg og utvidelser som hjelpeteknologi. Dette inkluderer stavekontroll, passordstøtte, støtte for tekst-til-tale og synkronisert visuel fremheving av teksten som leses.
