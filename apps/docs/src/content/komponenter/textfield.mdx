## Om komponenten

TextField (også kalt Input og Input field) er et skjemaelement som gir brukeren mulighet til å skrive inn korte tekster.

**Egnet til:**

- Når bruker skal fylle inn for eksempel navn, bruksenhetsnummer, adresse og annen informasjon som får plass på én linje

**Uegnet til:**

- Når brukeren bare kan skrive inn ett alternativ fra en forhåndsdefinert liste. Se [Select](https://grunnmuren.obos.no/komponenter/select), [RadioButton](https://grunnmuren.obos.no/komponenter/radiogroup) eller [Checkbox-group](https://grunnmuren.obos.no/komponenter/checkbox-group) i stedet.
- Når bruker blir bedt om å for eksempel beskrive noe. Bruk da heller [TextArea](https://grunnmuren.obos.no/komponenter/textarea)

## Formatering

Text-input har en fast høyde og brukes til å legge inn enklet innput i fritt format med en hvilken som helst kombinasjon av bokstaver, tall eller symboler.

### Anatomi

<Image src={img16} alt="" width={2000} height={845} />

1. **Label** Tekst som informerer brukeren om innholdet de må skrive inn i feltet. Det er påkrevd med mindre det foreligger et unntakstilfelle.
1. **Beskrivelse** Hjelpetekst som kan gi ekstra hjelp eller kontekst til brukeren. Brukes ofte for å forklare riktig dataformat.
1. **Verdi** Innholdet brukeren har skrevet inn i feltet.
1. **Field** Kontaineren der en bruker legger inn data. Må oppfylle 3:1 ikke-tekstkontrastkrav. (non-text contrast)

<Image src={img17} alt="8px spacer" width={672} height={8} />

### Format

Det er mulig å sette max- og min-lengde, samt å justere bredden på feltene:

```tsx live caption="Uncontrolled TextField med custom width og lengde"
<TextField
  description="Bruksenhetsnummeret, som tidligere ble kalt bolignummer, består av en bokstav og fire tall. F.eks. H0101"
  label="Bruksenhetsnummer"
  maxLength={5}
  minLength={5}
  size={5}
/>
```

<Image src={img18} alt="8px spacer" width={672} height={8} />

### Validering

Det er innebygget støtte for påkrevde felter:

```tsx live caption="Uncontrolled, påkrevd TextField"
<form onSubmit={(e) => e.preventDefault()} className="grid gap-6">
  <TextField isRequired label="Navn" />
  <Button type="submit" className="w-fit">
    Send inn
  </Button>
</form>
```

## Universell utforming

### Husk på

- Skjemaelementer burde ha en en-kolonnes layout, og hvert element skal ha sin egen label for å beskrive bruk. Om label av en eller annen grunn ikke skal vises skal den fortsatt finnes, men bruke klassen pkt-sr-only slik at skjemaet fortsatt er brukervennlig.
- Alle tekstfelt bør ha et tilknyttet label (ledetekst) eller instruksjon som viser hvordan skjemaet skal fylles ut, inkludert informasjon om obligatoriske felt.
- Label (ledeteksten) bør være koblet til tekstfeltet i koden slik at skjermlesere kan lese opp riktig informasjon. Komplekse skjema kan kreve mer hjelp enn bare enn labels.
- Bruker du symboler i tekstfeltet bør symbolene enten være logiske og kjente, eller så må de forklares før de brukes for å unngå forvirring.
- Feilmeldinger bør være kodet som tekst og identifisere det spesifikke skjemaelementet hvor feilen oppstod. Feilmeldingen må beskrive feilen og være synlig uten at brukeren gjør noen ekstra handlinger.
- Prefiks og suffiks er et ekstra visuelt hjelpemiddel, som blir ignorert av skjermlesere. Vi må alltid ha en beskrivende ledetekst. Prefiks og suffiks er plassert utenfor inndatafeltene de tilhører. Da unngår vi at de ikke skaper trøbbel i noen nettlesere, som kan sette inn et ikon i inndatafeltet

### Sjekkliste

✓ Alle skjemaelementene har tilknyttet label eller instruksjon

✓ Det er tydelig merket hvilke felter som er valgfrie

✓ Feilmeldinger er kodet som tekst og er i nærheten av det spesifikke tekstfeltet der feilen oppstod

✓ Feilmeldinger er tilgjengelig for skjermleser og keyboard-navigering

## Interaksjon

### Tastatur

| Key                                    | Interaksjon                                                                        |
| -------------------------------------- | ---------------------------------------------------------------------------------- |
| Tab                                    | Fører fokus til inputfield                                                         |
| Enter eller space                      | Åpner tilleggsfunksjoner som datepicker ved å aktivere kalender-ikonet             |
| Esc                                    | Lukker tilleggsfunksjoner                                                          |
| ctrl eller opt + venstre/høyre piltast | Beveger brukeren ord for ord i inputfeltet                                         |
| ctrl eller opt + opp/ned piltast       | Flytter brukeren til starten eller slutten av det som er skrevet inn i inputfeltet |
